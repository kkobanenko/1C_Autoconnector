# Анализ коллизий UI ↔ генератор SQL (15.03.2026)

## Симптом

- **UI:** 2 выбранных поля (1 в корневой, 1 в связанной таблице)
- **Генератор:** `ValueError: Нет выбранных полей`
- **Traceback:** `view_generator.py:603` — `if not self.selected_fields`

## Выявленные коллизии

### 1. Коллизия: excluded_fields не синхронизируются перед генерацией SQL

**Описание:** При нажатии «Сгенерировать SQL» в генератор передаётся `gen_excluded_fields` без синхронизации из чекбоксов `gen_f_*`. При сохранении конфигурации такая синхронизация есть (строки 2387–2403).

**Причина:** `_render_config_section` выполняется во fragment (`st.fragment`). Fragment может отработать асинхронно или позже основного скрипта. К моменту клика «Сгенерировать SQL» `gen_excluded_fields` может содержать устаревшие данные.

**Следствие:** Генератор получает `excluded_fields`, где все поля помечены как исключённые, и не добавляет ни одного поля в `selected_fields`.

### 2. Коллизия: разные источники истины для enabled/table_config

**Описание:** `table_config` синхронизируется из `gen_en_*` перед генерацией (строки 2686–2697), а `excluded_fields` — нет.

**Причина:** Несогласованность: для enabled делается явная синхронизация, для excluded — нет.

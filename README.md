# Генератор SQL VIEW для 1С

Генератор SQL представлений (VIEW) для нативной базы данных 1С с рекурсивными JOIN до указанного уровня вложенности.

## Возможности

- Парсинг структуры данных из .docx файла
- Автоматическое определение связей между таблицами через поля binary(16)
- Рекурсивная генерация JOIN (глубина вниз / вверх настраивается в мастере)
- Использование человеческих названий полей из документации
- Исправление искаженных дат (вычитание 2000 лет из дат >= 3000 года)
- Streamlit UI: мастер «Генератор VIEW», ручной режим, боковая панель «Контекст»
- **Секция 10:** автоматические сценарии заполнения конфигурации связей и полей (`utils/config_scenarios.py`) — в том числе вариант без самоссылок по рёбрам графа
- **Секция 10:** навигация по ссылочным полям с подсказкой пути от корневой таблицы
- CLI интерфейс для автоматизации

Подробное описание экранов, история изменений и требования: [docs/PRD.md](docs/PRD.md) (в т.ч. раздел 8.9).

## Установка

1. Установите зависимости:

```bash
pip install -r requirements.txt
```

## Использование

### Streamlit UI (рекомендуется)

Запуск (порт по умолчанию в [.streamlit/config.toml](.streamlit/config.toml) и [config.py](config.py) — **8512**):

```bash
streamlit run app.py --server.port 8512
```

Либо без указания порта, если он задан в `config.toml`:

```bash
streamlit run app.py
```

Приложение открывается по адресу: http://localhost:8512

В интерфейсе:

1. Подключитесь к базе данных (введите параметры подключения)
2. Выберите файл структуры (.docx)
3. Укажите название таблицы фактов (например, `Document653`)
4. Настройте параметры генерации
5. Постройте граф связей, при необходимости примените автосценарий в секции «Настройка таблиц и связей»
6. Нажмите «Генерация SQL» / сохраните конфигурацию по мастеру

### CLI интерфейс

```bash
python generate_view.py Document653 --max-depth 5 --output output/view.sql
```

Параметры:

- `fact_table` — название таблицы фактов (обязательно)
- `--structure-file` — путь к файлу структуры .docx (по умолчанию: `input/Структура.docx`)
- `--output` — путь к выходному SQL файлу
- `--max-depth` — максимальный уровень рекурсии (по умолчанию: 5)
- `--no-fix-dates` — не исправлять искаженные даты
- `--connection-string` — строка подключения к БД

## Формат файла структуры

Файл структуры должен быть в формате .docx и содержать описание таблиц в следующем формате:

```
Справочник.аксНастройкиЗагрузкиОбъектов => Reference37172, Основная
ID (Ссылка)
Version (ВерсияДанных)
Marked (ПометкаУдаления)
Fld37206 (ИмяОбъекта)
Fld37207 (ПредставлениеОбъекта)
```

## Тестирование

Для тестирования используйте таблицу `Document653`:

```bash
python generate_view.py Document653
```

## Версионирование

Перед коммитом изменений рекомендуется увеличить версию:

```bash
python bump_version.py
```

Скрипт увеличит минорную версию в `config.py` (например, 1.0.0 → 1.0.1).

## Примечания

- Таблицы в БД могут иметь префикс подчеркивания (`_Document653`) в зависимости от схемы
- Поля типа `binary(16)` используются для связей и по умолчанию не попадают в SELECT (настраивается в мастере)
- Поля типа `datetime2(0)` с датами >= 3000 года при опции «исправлять даты» могут корректироваться
- Максимальная глубина обхода графа задаётся в мастере (параметры «вниз» / «вверх»)
